<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argv -> backend.py | `--port` / `--ready-file` / `--cache-dir` / `--log-dir` values flow in from Tauri (Phase 3) or the test harness (Plan 03). argparse `type=int` / `type=Path` coerces. |
| Loopback HTTP (`127.0.0.1:<port>`) -> Flask app | In-process probe thread and (Phase 3) Tauri-hosted Vue frontend talk over loopback. Binding `127.0.0.1` (not `0.0.0.0`) is the only access-control layer. |
| Bundled `_MEIPASS` -> runtime env vars (`SSL_CERT_FILE`, `REQUESTS_CA_BUNDLE`) | Env-var set at startup determines which CA bundle downstream HTTPS calls trust. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Information Disclosure | `socketio.run` host binding | mitigate | Change `host='0.0.0.0'` -> `host='127.0.0.1'` (PITFALLS.md §Sec-7). Loopback-only bind prevents LAN devices from reaching the LCU bridge and extracting Riot auth tokens. Grep check in acceptance_criteria enforces. |
| T-01-06 | Elevation of Privilege | Werkzeug debug mode | mitigate | `debug=False` hardcoded (RESEARCH.md Pitfall #7). Werkzeug debugger PIN endpoint would allow arbitrary code execution from any local browser. |
| T-01-07 | Tampering (half-written ready-file misparses) | `_atomic_write_ready_file` | mitigate | `tmp + os.replace` atomic rename. `os.replace` is atomic on Windows NTFS and POSIX. Plan 03 test_backend_cli.py verifies the ready-file is well-formed JSON. |
| T-01-08 | Denial of Service (probe timeout leaves server alive with stale ready-file host believes) | `_probe_health_then_signal_ready` timeout path | mitigate | `os._exit(1)` on timeout — kills the process rather than only the probe thread (which `sys.exit(1)` would do). CI smoke test times out at 10 s catching this. |
| T-01-09 | Input Validation (out-of-range port) | argparse `--port` | accept | `type=int` coercion plus OS-level bind failure gives a non-zero exit within milliseconds. Adding range validation (0-65535) is low-value — misuse is dev-mode only; frozen bundle gets `--port` from Tauri which already allocates valid ports. ASVS V5 accepts this as low-risk given the localhost-only trust model. |
| T-01-10 | Information Disclosure (ready-file world-readable leaks port) | ready-file writes | accept | Windows per-user install puts the file in the user-private `%TEMP%` or pytest `tmp_path`. Phase 3 Tauri will place it in a Tauri-managed temp dir with user-private ACLs. Low risk on single-user Windows boxes. |
</threat_model>
