<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Source tree -> bundled .exe | What code/data lands inside PyInstaller's archive reaches every installed user. Spec-level `excludes` is the only chokepoint. |
| Bundle `_MEIPASS` temp dir -> user-writable dir | Read-only bundled resources (immutable per-launch) vs. cache/logs/state (persistent, user-private). Mixing them is Pitfall #6. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | `backend.spec` bundling | mitigate | `excludes=['supabase', 'gotrue', 'postgrest', 'realtime', 'storage3', 'supabase_functions', 'supabase_auth']` prevents the supabase-py client and its transitives from landing in the `.exe`; Plan 03 CI adds `strings dist/backend.exe \| grep -i supabase` as a fail-the-build check. Closes the "SUPABASE_SERVICE_ROLE_KEY extractable from binary" threat even if the key reaches a developer's `.env` during dev mode. |
| T-01-02 | Tampering (AV false-positive quarantine surfaces as DoS for end users) | `backend.spec` / EXE block | mitigate | `upx=False` (Pitfall #2) is locked at spec level. Plan 03 CI adds a grep guard (`grep -E '^\s*upx\s*=\s*True' backend.spec && exit 1`) so no future PR can flip it silently. |
| T-01-03 | Denial of Service (stale TLS root CA bundle) | `certifi` bundling | accept | Phase 4 release checklist owns the `pip install -U certifi` cadence. Phase 1 pins `certifi` transitively via `requests`/`urllib3` and bundles `certifi.where()` at spec-build time — stale bundles are a release-process problem, not a Phase 1 code problem. |
| T-01-04 | Information Disclosure (path-resolution bugs writing secrets to ephemeral `_MEIPASS` then losing them / leaving them on disk) | `resources.py` | mitigate | Three-function split (`user_cache_dir` / `user_log_dir` / `user_data_dir`) makes the read-only vs writable distinction impossible to confuse at call sites. D-14 grep-enforces no `__file__`/`cwd()` after Phase 1. Plan 02 removes the last `__file__` reference. |
